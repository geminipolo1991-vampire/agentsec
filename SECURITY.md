# Security policy

Do not report vulnerabilities by opening a public issue. Until a private hosting
contact is configured, retain the report and contact the repository owner through
an agreed private channel. Do not include credentials, personal data, production
prompts, or exploitable customer evidence in a report.

This repository is a research PoC. The local HMAC signer, in-memory stores, mock
containment, recorded Codex analysis, and single-node deployment scaffold are not
production security controls. See the documented limitations before testing.
